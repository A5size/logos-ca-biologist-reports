# LOGOS-CA Biologist Reports

An interactive viewer for fictitious biological field reports generated by a Large Language Model (LLM) through a cellular automaton simulation.

**[→ View Live](https://a5size.github.io/logos-ca-biologist-reports/)**

![screenshot](screenshot.png)

## Overview

Each cell in a grid represents a biologist's field report — complete with a title, summary, and a pixel-art sketch of the observed organism or biome. At every simulation step, each report is rewritten by an LLM based on the previous report and those of neighboring cells.

The viewer supports multiple experiments with different grid sizes, models, sketch resolutions, and step counts. Experiments are selected from a terminal-style menu on launch.

### Example: Experiment 04-03 (10×10, GPT-5-mini)

The grid is divided into four weakly-coupled 5×5 regions, connected only through a few boundary cells. Over 50 steps, each region independently develops its own "ecological" vocabulary and visual style — an emergent island biogeography driven entirely by language.

| Region | Location | Emergent Theme |
|--------|----------|----------------|
| R1 | Top-left | Lamina — radiant leaf-like structures in montane cloud forests |
| R2 | Top-right | Mucilage — gelatinous organisms in mangrove intertidal zones |
| R3 | Bottom-left | Pulsatilis — pulsing, resonant spiral forms |
| R4 | Bottom-right | Mosaics — fog-harvesting capillary condensate systems |

> **Note:** These themes were not designed or specified by humans in any way. The only input was a neutral placeholder report identical across all cells. The themes above are labels we assigned *after the fact* to describe the tendencies that emerged spontaneously from the LLM-driven simulation.

## Background

This project extends the [LOGOS-CA](https://github.com/A5size/LOGOS-CA) framework proposed in:

> Utimula, K. (2026). **LOGOS-CA: A Cellular Automaton Using Natural Language as State and Rule.** arXiv preprint.
> [https://arxiv.org/abs/2602.00036](https://arxiv.org/abs/2602.00036)

LOGOS-CA is a cellular automaton where both cell states and update rules are described in natural language, with state transitions determined by an LLM. The original paper demonstrated this with forest fire simulations and artificial life experiments.

This project adapts the framework so that each cell state is a structured biological report (title, summary, and pixel-art sketch), and the LLM acts as a field biologist fabricating new observations influenced by nearby reports.

## Usage

### Viewer Controls

| Input | Action |
|-------|--------|
| Slider / ◀ ▶ buttons / ← → keys | Switch simulation step |
| Space | Play / Pause auto-advance |
| 1x / 2x / 4x button | Change playback speed |
| Hover over cell (PC) / Tap cell (mobile) | Preview report |
| Click cell | Pin report (click again or Esc to unpin) |
| ◀ EXP button | Return to experiment selection |

### Share

Each report can be shared via the buttons at the top of the detail panel:

- **𝕏 POST TO X** — copies a share card image and opens X with pre-filled text
- **☐ COPY IMAGE** — copies the share card to clipboard
- **↓ DOWNLOAD** — downloads the share card as PNG

Share card images include the experiment ID, step number, cell position, and (for 10×10 grids) the region badge.

## Adding Experiments

### 1. Convert simulation data

Use `convert_data.py` to convert raw simulation output into the compact viewer format and register it in the manifest:

```bash
pip install Pillow

# Basic usage
python convert_data.py \
  --id 04-03 \
  --desc "10x10 grid, 4 weakly-coupled 5x5 regions. GPT-5-mini, 50 steps." \
  biologist_reports_result.json

# With custom output directory
python convert_data.py \
  --id 02-01 \
  --desc "Preliminary. 11x11 field, GPT-5-mini." \
  --outdir ./public \
  biologist_reports_result.json

# With non-standard sketch field name
python convert_data.py \
  --id 03-01 \
  --desc "High-res sketch test. 32px sketches." \
  --sketch-key sketch_rgb_32x32 \
  biologist_reports_result.json
```

This produces:
- `exp-{id}-data.json` — compact experiment data (base64 PNG sketches, minified JSON)
- `experiments.json` — manifest file (created automatically, updated on each run)

The sketch field name (`sketch_rgb_16x16`, `sketch_rgb_32x32`, etc.) is auto-detected. Grid size, model name, and step count are read from the simulation metadata.

### 2. Deploy

Place `index.html`, `experiments.json`, and all `exp-*-data.json` files in the same directory and serve via any static file server.

```
public/
├── index.html
├── experiments.json          # auto-generated manifest
├── exp-04-03-data.json       # experiment data
├── exp-02-01-data.json
└── ...
```

### Manifest format

`experiments.json` is a JSON array managed automatically by `convert_data.py`. Each entry contains:

```json
{
  "id": "04-03",
  "desc": "10x10 grid, 4 weakly-coupled 5x5 regions. GPT-5-mini, 50 steps.",
  "file": "exp-04-03-data.json",
  "grid_w": 10,
  "grid_h": 10,
  "model": "gpt-5-mini",
  "steps": 51,
  "sketch_size": 16
}
```

Running `convert_data.py` with the same `--id` will update the existing entry rather than creating a duplicate.

## Viewer Features

- **Variable grid sizes**: automatically adjusts cell sizes and layout for any grid dimensions
- **Region display**: 10×10 grids show the 4 weakly-coupled region borders and badges; other sizes display a plain grid
- **Variable sketch sizes**: renders any square sketch resolution (8×8, 16×16, 32×32, etc.) with pixelated scaling
- **Mobile support**: tap-to-select with download/share buttons accessible without pinning
- **Responsive**: three-stage responsive layout (desktop / tablet / phone)
- **CRT aesthetic**: scanline overlay, retro fonts, terminal boot sequence
- **OFUSE support link**: integrated in intro, experiment selection, and status bar

## Simulation

The code used to run the simulations is included in the [`simulation/`](simulation/) directory. See [`simulation/README.md`](simulation/README.md) for setup and usage instructions.

A typical workflow:

```bash
# 1. Run simulation
cd simulation
pip install -r requirements.txt
export OPENAI_API_KEY="your-key"
python biologist_reports.py

# 2. Convert results for the viewer
cd ..
pip install Pillow
python convert_data.py \
  --id 04-03 \
  --desc "10x10 grid, GPT-5-mini, 50 steps." \
  simulation/biologist_reports_result.json

# 3. Serve locally
python -m http.server 8000
```

> **⚠️ Cost Warning:** Simulations use the OpenAI API and incur charges. A 10×10 grid for 50 steps makes 5,000 API calls. Start small to estimate costs.
